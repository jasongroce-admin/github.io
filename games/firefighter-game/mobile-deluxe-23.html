Builder Help

Core Build Flow

  • Set your Level Name, then place models from the Asset Library onto the canvas to start building.
  • Use the left Asset Library to pick or upload images, then click canvas once to place.
  • Use Layers to move, scale, flip, lock physics, set burnability, and tune burn boxes.
  • The fire truck starts on the Left. Change Truck Side to Right anytime.
  • Burn boxes start on Easy so every burnable object works immediately; raise box tiers to Medium/Hard if wanted.
  • Use Test Difficulty + Quick Play to test easy/medium/hard behavior.
  • Use Events for tankers, tornado timing, and bonus vehicles. Save/Package from Export and Test.

What Each Export/Test Button Does

  • Load Current Level: loads the currently active game level into builder for editing.
  • New Blank: starts a fresh blank builder level and keeps your current settings defaults.
  • Save Local: saves this level to this browser on this computer (local only).
  • Load Local: reloads one of your browser-saved local builder levels.
  • Package: downloads a portable level package for later re-import into builder.
  • Load Package: imports a previously packaged JSON level into builder.
  • Share: copies level package JSON text to clipboard for sending.
  • Save Submission File: creates the merge-ready `.kvfdsubmission.json` for source updates.
  • Print: prints the level summary/integration information.
  • Test Level: launches quick test play using selected test difficulty.
  • Save To Game Slot: writes to local browser game slot override on this computer only.
  • Save Offline Folder: exports game + images + level for local offline player use.
  • Save Full Game Offline: exports game + images without attaching current custom level.

Permanent Source Update Workflow (Local Repo -> Online)

Use this when you want your builder changes to become real source files and then publish online after testing.

  • 1) Build and test level in builder until it plays correctly.
  • 2) Click Save Submission File and save the `.kvfdsubmission.json` file.
  • 3) Run tools\\apply-builder-submission-local-qa.cmd for local apply + test + Commit/Undo buttons.
  • 4) Select your submission file. The tool updates local source slot data in `mobile-deluxe-23.html`, waits for testing, then commits if approved.
  • 5) Optional fast path: run tools\\apply-builder-submission-and-publish.cmd for direct one-step apply + push.
  • 6) Refresh the site/game and verify the slot behavior online.

Important: Save To Game Slot is for fast local testing. Submission + local QA script is the safe permanent update path.
